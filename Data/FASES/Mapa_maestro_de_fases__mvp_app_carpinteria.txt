FASE 2A — VALIDACION DE ARQUITECTURA (EXPO VS BARE)
================================================================================
Objetivo: Evaluar si migrar a bare React Native o mantener Expo con prebuild.

Decision tomada: MANTENER EXPO con Expo prebuild / development builds.
Razon: Expo SDK 54 soporta expo-sqlite y react-native-mmkv via prebuild
sin necesidad de eject. El eject no existe como concepto en Expo moderno.

Entregables:
- Analisis documentado de 3 opciones (A: Expo Go, B: Expo prebuild, C: bare RN)
- Decision firme: Opcion B
- Documento Fase_2_BBDD_local_y_persistencia_v2_EXPO.txt adaptado

Dependencias: Fase 1
Estado: HECHA (integrada en Fase 2 v2)

FASE 3 — MOTOR DE LOGICA DEL PRODUCTO
================================================================================
Objetivo: Construir la inteligencia funcional local: generar proyectos DIY,
estimar materiales, optimizar cortes, recomendar tiendas y comparar precios.

Entregables:
- diyGenerator.ts mejorado:
  - 9 templates (estanteria, mesa, caja, banco, perchero, marco,
    soporte_plantas, zapatero + default)
  - Cada template con: pasos, materiales, herramientas, dificultad,
    tiempo estimado, summary
  - Keyword matching inteligente (KEYWORD_MAP)
  - Tipo ProjectDifficulty: 'easy' | 'medium' | 'hard'
- materialsGenerator.ts:
  - Estima tableros, tornillos (~8 por union), cola, tapones, cinta de canto
  - Escuadras si >4 piezas
- cuttingOptimizer.ts:
  - Algoritmo FFD (First Fit Decreasing)
  - Ordena piezas por area descendente
  - Prueba ambas orientaciones
  - Calcula: totalBoards, efficiency, totalWaste, detalle por tablero
- storeRecommender.ts:
  - 5 tiendas mock: Amazon, AliExpress, Leroy Merlin, Bricomart, ManoMano
  - Precios estimados por tipo de material
  - Multiplicadores por tienda
- priceComparator.ts:
  - Ranking compuesto: precio (40%) + rapidez (30%) + score base (30%)
  - Devuelve best, cheapest, fastest + recommendation textual
  - Tipo ComparisonResult exportado
- projectEngine.ts (orquestador):
  - runDIYProject(): idea → steps + materials + tools + stores + comparison
  - runProProject(): pieces → optimization + materials + stores + comparison
- Tipos nuevos en models/index.ts:
  - ProjectDifficulty, ComparisonResult, FullDIYResult, FullProResult
  - DIYResult ampliado con summary, difficulty, estimatedTime

Archivos clave:
- src/services/diyGenerator.ts
- src/services/materialsGenerator.ts
- src/services/cuttingOptimizer.ts
- src/services/storeRecommender.ts
- src/services/priceComparator.ts
- src/services/projectEngine.ts
- src/models/index.ts (ampliado)

  - src/routes/analytics.ts (POST /analytics/events)
  - src/services/openaiClient.ts (singleton OpenAI)
  - src/services/diyAiService.ts (structured output con json_schema)
  - src/services/proAiService.ts (tool calling loop, max 5 iteraciones)
  - src/schemas/diySchema.ts (JSON schema + system prompt DIY)
  - src/schemas/proSchema.ts (system prompt PRO + TOOL_DEFINITIONS)
  - src/tools/optimizeCuts.ts (replica logica local)
  - src/tools/estimateMaterials.ts (replica logica local)
  - src/tools/compareStoreOptions.ts (replica logica local)
  - src/tools/toolRegistry.ts (executeTool dispatcher)
  - src/utils/env.ts (OPENAI_API_KEY, PORT)

  - Lee API URL de app.config.ts via expo-constants
  - Funciones: generateDIYWithAI, generateProPlanWithAI, checkBackendHealth

FASE 5 — INTEGRACION END-TO-END EN LA APP
================================================================================
Objetivo: Conectar UI + backend + storage + fallback en flujos reales
con gestion de estado robusta.

Entregables:
- Hook useAsyncAction:
  - Estados: idle, loading, success, error
  - Proteccion contra doble pulsacion (lockRef)
  - Reset
- Hook useDIYWorkflow:
  - Orquesta: IA (con fallback local) → persistencia SQLite → resultado
  - Devuelve source ('ai' | 'local') para indicador visual
- Hook useProWorkflow:
  - Orquesta: IA con tools (con fallback local) → persistencia completa
  - Guarda proyecto, piezas, materiales, optimizacion en SQLite
- Componentes de estado:
  - LoadingState: spinner + mensaje
  - ErrorState: icono + mensaje + boton retry
  - SourceBadge: indica si resultado es IA o local
- Pantallas actualizadas:
  - DIYInputScreen: usa useDIYWorkflow, ActivityIndicator, fallback con Alert
  - ProInputScreen: usa useProWorkflow, persiste todo al optimizar
  - ProOptimizationScreen: persiste optimizacion y materiales en SQLite
  - ShopScreen: persiste tiendas en SQLite via useEffect

Archivos clave:
- src/hooks/useAsyncAction.ts
- src/hooks/useDIYWorkflow.ts
- src/hooks/useProWorkflow.ts
- src/components/LoadingState.tsx
