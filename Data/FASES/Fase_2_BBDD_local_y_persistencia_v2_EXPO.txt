FASE 2 — BBDD LOCAL Y PERSISTENCIA (OPCIÓN B: EXPO + PREBUILD)

DECISIÓN ARQUITECTÓNICA
Se mantiene Expo (SDK 54). No se migra a bare React Native.
Se usa Expo con development builds (prebuild) para acceder a módulos nativos.

STACK DE PERSISTENCIA
- expo-sqlite       → datos estructurados (proyectos, piezas, materiales, optimizaciones, tiendas)
- react-native-mmkv → preferencias ligeras (modo, último proyecto, unidades, onboarding)

LIBRERÍAS A INSTALAR
yarn:
  npx expo install expo-sqlite
  npx expo install react-native-mmkv

PREPARACIÓN DEL ENTORNO
1. npx expo install --fix              → corregir dependencias desactualizadas
2. npx expo install expo-sqlite        → SQLite oficial de Expo
3. npx expo install react-native-mmkv  → MMKV para preferencias
4. npx expo prebuild                   → genera ios/ y android/
5. npx expo run:ios (o run:android)    → compilar dev build

ESQUEMA SQLITE

TABLE projects
  id TEXT PRIMARY KEY
  name TEXT NOT NULL
  mode TEXT NOT NULL CHECK(mode IN ('diy','pro'))
  description TEXT DEFAULT ''
  created_at TEXT NOT NULL
  updated_at TEXT NOT NULL

TABLE pieces
  id TEXT PRIMARY KEY
  project_id TEXT NOT NULL REFERENCES projects(id) ON DELETE CASCADE
  width REAL NOT NULL
  height REAL NOT NULL
  quantity INTEGER NOT NULL DEFAULT 1
  thickness REAL NOT NULL DEFAULT 18

TABLE materials
  id TEXT PRIMARY KEY
  project_id TEXT NOT NULL REFERENCES projects(id) ON DELETE CASCADE
  name TEXT NOT NULL
  quantity REAL NOT NULL
  unit TEXT DEFAULT 'ud'

TABLE optimizations
  id TEXT PRIMARY KEY
  project_id TEXT NOT NULL REFERENCES projects(id) ON DELETE CASCADE
  board_width REAL NOT NULL
  board_height REAL NOT NULL
  total_boards INTEGER NOT NULL
  waste_percentage REAL NOT NULL
  efficiency REAL NOT NULL
  created_at TEXT NOT NULL

TABLE shop_options
  id TEXT PRIMARY KEY
  project_id TEXT NOT NULL REFERENCES projects(id) ON DELETE CASCADE
  store_name TEXT NOT NULL
  type TEXT NOT NULL CHECK(type IN ('online','physical'))
  price REAL NOT NULL
  delivery_time TEXT NOT NULL
  score REAL NOT NULL

MMKV KEYS
  @diy_selected_mode     → 'diy' | 'pro'
  @diy_last_project_id   → string
  @diy_user_units        → 'cm' | 'mm' | 'in'
  @diy_has_seen_onboarding → boolean

CRUD MÍNIMO POR REPOSITORIO

projectRepository:
  - createProject(project) → void
  - getProjects() → Project[]
  - getProjectById(id) → Project | null
  - updateProject(id, data) → void
  - deleteProject(id) → void

pieceRepository:
  - createPiece(piece) → void
  - getPiecesByProject(projectId) → Piece[]
  - deletePiecesByProject(projectId) → void

materialRepository:
  - createMaterial(material) → void
  - getMaterialsByProject(projectId) → Material[]
  - deleteMaterialsByProject(projectId) → void

optimizationRepository:
  - saveOptimization(optimization) → void
  - getOptimizationByProject(projectId) → Optimization | null

shopRepository:
  - saveShopOptions(projectId, options[]) → void
  - getShopOptionsByProject(projectId) → StoreOption[]

settingsStorage:
  - getSelectedMode() → 'diy' | 'pro'
  - setSelectedMode(mode) → void
  - getLastProjectId() → string | null
  - setLastProjectId(id) → void
  - getUserUnits() → 'cm' | 'mm' | 'in'
  - setUserUnits(units) → void
  - getHasSeenOnboarding() → boolean
  - setHasSeenOnboarding(value) → void
