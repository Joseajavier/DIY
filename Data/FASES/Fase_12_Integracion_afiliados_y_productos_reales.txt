FASE 12 — INTEGRACION DE AFILIADOS Y PRODUCTOS REALES

OBJETIVO
Reemplazar las tiendas mock por productos reales de Amazon (y opcionalmente Leroy Merlin / ManoMano) usando programas de afiliados. El usuario ve productos reales con foto, precio, enlace de compra. Si compra, la app gana comision.

PREREQUISITOS
Antes de implementar esta fase, el propietario debe:
1. Registrarse en Amazon Associates España: affiliate-program.amazon.es
2. Esperar aprobacion (1-2 dias)
3. Obtener: Associate Tag (ej: diyapp-21)
4. Activar PA-API 5.0 en la consola de Associates
5. Obtener: Access Key + Secret Key
6. Guardar las 3 credenciales en el .env del backend

CREDENCIALES NECESARIAS EN BACKEND .env
AMAZON_ASSOCIATE_TAG=diyapp-21
AMAZON_ACCESS_KEY=AKIA...
AMAZON_SECRET_KEY=xxxxx...
AMAZON_MARKETPLACE=www.amazon.es
AMAZON_REGION=eu-west-1

ARQUITECTURA

App Expo
→ llama al backend (sin cambios en la app, misma estructura)

Backend
→ nuevo servicio: amazonProductService.ts
→ busca productos en Amazon PA-API 5.0
→ devuelve: titulo, precio, foto, enlace afiliado, valoraciones, disponibilidad
→ cache de resultados (1h) para no abusar de la API

ENDPOINTS NUEVOS O MODIFICADOS

POST /shop/search
Input:
  - query: string (ej: "tablero melamina 244x122cm")
  - category: string (opcional, ej: "Tools", "DIY")
  - country: string (ej: "es", "uk", "de")
Output:
  - products[]:
    - title: string
    - price: number
    - currency: string
    - imageUrl: string
    - productUrl: string (con tag afiliado)
    - rating: number
    - reviewCount: number
    - availability: string
    - source: "amazon"

POST /shop/materials-search
Input:
  - materials[]: { name, quantity, unit }
  - country: string
Output:
  - results[]:
    - materialName: string
    - products[]: (mismo formato que arriba)
    - bestMatch: product (el mas relevante)

LOGICA DE BUSQUEDA
Para cada material de la lista:
1. Generar query de busqueda inteligente
   - "Tablero melamina 244x122cm" → buscar tal cual
   - "Tornillos confirmat 5x50mm" → buscar "tornillos confirmat 5x50"
   - "Cola de carpintero" → buscar "cola carpintero madera"
2. Buscar en Amazon PA-API con SearchItems
3. Filtrar por categoria relevante (DIY, Tools, Industrial)
4. Ordenar por relevancia + precio
5. Devolver top 3 resultados por material
6. Cachear resultado por query + pais (1 hora)

MULTI-PAIS
Detectar pais del usuario por:
1. Setting de idioma en la app (es → amazon.es, en → amazon.co.uk)
2. O setting explicito en SettingsScreen

Marketplaces Amazon:
- es → www.amazon.es (eu-west-1)
- uk → www.amazon.co.uk (eu-west-1)
- fr → www.amazon.fr (eu-west-1)
- de → www.amazon.de (eu-west-1)
- it → www.amazon.it (eu-west-1)
- us → www.amazon.com (us-east-1)

MONETIZACION
- Comision Amazon Associates: 1-10% segun categoria
- Herramientas/DIY suele ser 3-5%
- Si un usuario compra 50€ en materiales → 1.50-2.50€ de comision
- Con 1000 usuarios activos que compren 1 vez/mes → 1500-2500€/mes

RESTRICCIONES
- No guardar precios en SQLite local (cambian constantemente)
- Cachear solo en backend (max 1 hora)
- No mostrar precio si no viene de la API (evitar datos obsoletos)
- Si API falla → fallback a mock con mensaje "Precios no disponibles, mostrando estimacion"
- Respetar limites de Amazon PA-API (1 request/segundo)

IMPORTANTE

El propietario debe registrarse primero y proporcionar:
- Associate Tag
- Access Key
- Secret Key

Una vez tenga las credenciales, se puede implementar en 3-5 dias.
