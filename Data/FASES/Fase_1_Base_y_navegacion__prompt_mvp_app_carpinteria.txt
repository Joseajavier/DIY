FASE 1 — BASE DEL PROYECTO Y NAVEGACIÓN

Quiero que actúes como un arquitecto y desarrollador senior de React Native con TypeScript.

Necesito que me ayudes a construir un MVP de una app móvil de carpintería y bricolaje. La app debe estar pensada para dos tipos de usuario:

1. DIY MODE (principiante)
2. PRO MODE (avanzado)

OBJETIVO DEL PRODUCTO
La app debe ayudar a:
- crear proyectos DIY en casa
- calcular materiales y herramientas
- generar pasos de montaje
- optimizar cortes de madera
- recomendar dónde comprar materiales
- comparar opciones de compra

ALCANCE DEL MVP
Quiero un MVP SIN:
- backend
- autenticación
- APIs de IA
- 3D

Quiero un MVP CON:
- lógica local
- datos mock
- navegación completa
- flujo funcional de principio a fin

STACK OBLIGATORIO
- React Native
- TypeScript
- React Navigation
- estado local
- usar Yarn, no npm

ARQUITECTURA BASE
Quiero esta estructura:

src/

  services/
  models/
  utils/
  navigation/

FLUJOS

DIY MODE
El usuario describe un proyecto sencillo y la app devuelve:
- pasos de trabajo
- materiales
- herramientas
- recomendaciones de compra mock

PRO MODE
El usuario introduce medidas exactas y la app devuelve:
- piezas
- optimización de cortes
- eficiencia de material
- materiales estimados
- recomendaciones de compra mock

SERVICIOS QUE QUIERO IMPLEMENTAR
Crea estos servicios con lógica sencilla pero útil:

1. diyGenerator.ts
- convierte una idea simple en pasos
- propone materiales
- propone herramientas

2. materialsGenerator.ts
- calcula tableros
- estima tornillos (aprox. 8 por unión cuando tenga sentido)
- contempla cola

3. cuttingOptimizer.ts
- ordena piezas de mayor a menor
- intenta encajarlas en tableros
- minimiza desperdicio
- enfoque simple tipo Cutting Stock Problem para MVP

4. storeRecommender.ts
- devuelve opciones mock de tiendas
- Amazon = rápido
- AliExpress = barato
- Leroy Merlin = tienda física

5. priceComparator.ts
- compara precio
- compara plazo de entrega
- calcula una puntuación simple de recomendación
