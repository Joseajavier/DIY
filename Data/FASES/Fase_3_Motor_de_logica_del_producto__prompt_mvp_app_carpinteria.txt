LO QUE QUIERO QUE IMPLEMENTES

1. diyGenerator.ts
Quiero un generador local y determinista que reciba una idea o descripción sencilla de proyecto DIY y devuelva:
- nombre sugerido del proyecto
- breve descripción normalizada
- lista de pasos
- lista de materiales estimados
- lista de herramientas sugeridas
- dificultad estimada: 'easy' | 'medium' | 'hard'
- tiempo estimado en texto: por ejemplo '2-3 horas'

Este generador no tiene que ser IA real.
Quiero lógica local razonable basada en reglas simples y palabras clave.

Ejemplos de entradas:
- "quiero hacer una balda de pared"
- "mesa auxiliar pequeña"
- "estantería sencilla para libros"
- "soporte para plantas"

Quiero que la salida sea útil y consistente, aunque sea heurística.

2. materialsGenerator.ts
Quiero un servicio que, a partir de piezas o de un tipo de proyecto, calcule materiales base.
Debe poder:
- estimar tableros
- estimar listones si aplica
- estimar tornillos
- estimar cola
- estimar cantos o acabado básico si procede
- devolver materiales con cantidad y unidad

Reglas mínimas:
- aproximación simple pero coherente
- tornillos estimados con una regla clara
- cola como cantidad estimada razonable
- no inventar cálculos absurdos

3. cuttingOptimizer.ts
Quiero un optimizador local simple de cortes para MVP.
Debe:
- recibir piezas con width, height, quantity, thickness
- expandir cantidades en piezas individuales
- ordenar piezas de mayor a menor
- asignarlas a tableros
- intentar minimizar desperdicio
- calcular:
  - totalBoards
  - usedArea
  - wasteArea
  - wastePercentage
  - efficiency
- devolver también el detalle de distribución por tablero

No necesito un algoritmo perfecto.
Necesito uno simple, explicable y útil para MVP.
Puede ser un algoritmo heurístico tipo first-fit decreasing o similar.

IMPORTANTE
Quiero que el resultado sea estable y legible.
No quiero magia negra ni un algoritmo difícil de mantener.

4. storeRecommender.ts
Quiero un servicio local mock que genere opciones de tienda según el tipo de compra.

Tiendas base:
- Amazon → rápido
- AliExpress → barato
- Leroy Merlin → física / bricolaje

Puede incluir alguna más si tiene sentido, pero no muchas.

Debe devolver para cada opción:
- id
- projectId si aplica
- storeName
- type
- price
- deliveryTime
- score

La lógica puede basarse en:
- rapidez
- precio
- conveniencia según el proyecto
- si el usuario está en modo DIY o PRO

5. priceComparator.ts
Quiero un servicio que reciba varias StoreOption y calcule:
- ranking final
- mejor opción por precio
- mejor opción por rapidez
- mejor opción global
- explicación corta del motivo de la recomendación

Debe generar una puntuación clara y simple.
No quiero una fórmula absurda.
Quiero algo fácil de entender y mantener.
