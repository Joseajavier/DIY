IMÁGENES
Para las imágenes de herramientas:
- NO podemos usar fotos de Amazon (copyright)
- NO podemos usar fotos de las marcas (copyright)

OPCIONES:
A) Usar iconos/emojis por tipo (lo que tenemos ahora) — FÁCIL pero feo
B) Usar imágenes genéricas de herramientas de Unsplash/Pexels — LEGAL pero poco específicas
C) Usar la API de Amazon Associates (Fase 12) para obtener imágenes — LEGAL y específicas
   → Las imágenes de productos que devuelve la PA-API se pueden mostrar mientras
     se enlace al producto. Es parte del programa de afiliados.
D) Generar iconos SVG simples por tipo de herramienta — PROPIO y escalable

RECOMENDACIÓN: Combinación C + D
- Ahora mismo: iconos SVG o ilustraciones simples por tipo (Fase 16)
- Cuando tenga Amazon Associates: fotos reales del producto (Fase 12)

DISEÑO DE CARDS (como pide el usuario)
El usuario quiere que se parezca al estilo Amazon:
- Imagen grande a la izquierda
- Nombre completo del producto
- Badge de tipo (Cable eléctrico / Batería)
- Estrellas / valoración (mock por ahora, real con API)
- Precio grande y visible
- Entrega estimada (mock)
- Botón "Añadir a la cesta" / "Ver en Amazon"

AGRUPACIÓN POR SECCIONES
El usuario quiere ver por secciones, no todo mezclado:
- Header de sección: "🪚 Sierras circulares" con número de resultados
- Debajo las cards de esa sección
- Luego "🪚 Sierras de inmersión" etc.
- Las secciones se filtran con los chips de categoría

ESTRUCTURA DE DATOS MEJORADA
Cada producto debe tener:
- id, typeId, brandId, model
- tier, use[], power
- priceMin, priceMax
- description (1-2 líneas)
- features[] (specs clave)
- bestFor (1 línea)
- rating: number (1-5, mock)
- reviewCount: number (mock)
- imageUrl?: string (para futuro)
